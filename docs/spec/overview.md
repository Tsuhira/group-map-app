# グループマップアプリ 全体仕様

Amway などのネットワークビジネスの系統図を可視化・管理するアプリ。

---

## テックスタック

| 項目 | 内容 |
|------|------|
| フロントエンド | React 19 + Vite |
| ツリー描画 | D3.js（放射状レイアウト） |
| ホスティング | GitHub Pages（GitHub Actions で自動デプロイ） |
| データ保存 | Firestore REST API（kuma-app 経由）/ sampleNodes（スタンドアロン） |
| 認証 | Firebase Auth REST API（kuma-app SSO、SDK 不使用） |
| UI アイコン | lucide-react |

---

## データモデル

### ノード（Node）

```ts
{
  id: string           // ランダム UUID
  name: string         // 表示名（同一親配下での重複を許容）
  parentId: string | null  // 親ノードの id（ルートは null）
  pinLevel: string     // ピンレベル（任意文字列）
  active: boolean      // アクティブ状態
  joinDate: string     // 入会日（YYYY-MM-DD）
  note: string         // 備考
  userId: string | null  // ノードを「自分」と主張したユーザーの uid
}
```

### ピンレベル（pinLevel）

- 任意の文字列で自由に設定する（事前定義なし）
- `name+rank` モードでノード名の下段に小さく表示される

## 表示ルール

### ノードの見た目
- 形: **横長の楕円形**（`<ellipse>`）
- アクティブノード: rx=70px、ry=28px（2行ラベル時 ry=36px）
- **非アクティブノード**: rx=ry=28px（円形・テキストなし・opacity 0.5）
- 色: アクティブ `#1e4470`（ネイビー）/ 非アクティブ `#1a2a3a`
- ラベル: 名前（デフォルト）/ 名前＋ピンレベル（設定で切り替え）

### 接続線
- 親子間を直線で接続（`<line>` 要素）
- アクティブノードへの線: 実線 `rgba(255,255,255,0.25)`
- 非アクティブノードへの線: 破線 `rgba(255,255,255,0.08)` / `stroke-dasharray: 5 4`
- 線の太さ: 一律 1.5px

### レイアウト
- D3 の `d3.tree().size([2π, R])` を使用した**放射状（Radial）レイアウト**
- 規定ノードを中心に、子ノードが同心円状に広がる
- レベル間隔: 160px（`LEVEL_RADIUS = NODE_RX * 2 + MIN_GAP`）

---

## 規定ノード（サブツリー表示）

- 規定ノード: マップに表示するサブツリーのルートとなるノード
- デフォルトはデータ全体のルートノード
- ノードを右クリック → コンテキストメニュー「規定ノードに設定」で変更
- パンくずに現在の規定ノードまでの階層を表示
- 「↑ 上の階層へ」で親ノードを規定に
- 「全体表示」でルートに戻す
