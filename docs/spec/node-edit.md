# ノード追加・編集・削除 仕様

## サイドパネル（Sidebar）

ノードをクリックすると表示。デスクトップは右側、モバイルは画面下部からスライドイン。

### 表示内容（詳細モード）
- ノード名
- ピンレベルバッジ（設定されている場合）
- アクティブ状態
- 入会日
- 備考
- 「あなたのノード」バッジ（`userId` が自分の uid と一致する場合）
- 「編集」ボタン
- 「子ノードを追加」ボタン
- 「削除」ボタン（子ノードがない場合のみ有効）

### 編集フォーム
| フィールド | 入力方式 |
|-----------|---------|
| 名前 | テキスト入力（必須） |
| 親ノード | 読み取り専用表示（変更不可） |
| ピンレベル | テキスト入力（任意） |
| アクティブ | トグルスイッチ |
| 入会日 | 日付ピッカー |
| 備考 | テキストエリア |

Firestore モード時のみ: 「このノードは自分」ボタンで `userId` に自分の uid をセット。

### 子ノード追加
- 「子ノードを追加」ボタン → 新規ノードの編集フォームを開く
- 親ノード欄に選択中ノードの名前が自動入力される

---

## ルートノード追加

- ノードが 0 件の場合、空状態オーバーレイに「＋ 最初のノードを作成」ボタンを表示
- 右クリックメニューの「ルートを追加」からも作成可能
- `parentId: null` でルートノードとして作成

---

## ノード削除

- 子ノードを持つノードは削除不可（ボタンを disabled にして理由を表示）
- 削除前に確認ダイアログを表示
- 削除後、親ノードを自動選択

---

## キャンバス上での操作

- ノードを右クリック → コンテキストメニュー
  - 「子ノードを追加」
  - 「規定ノードに設定」
  - 「編集」
  - 「削除」

---

## バリデーション

- 名前: 空文字不可（同一親配下での重複は許容）
- 入会日: 未来日不可

---

## ID 管理

- ノードの識別は `id`（`crypto.randomUUID()` で生成）で行う
- 表示名の重複は許容（同じ名前のノードを別々に作成可能）
